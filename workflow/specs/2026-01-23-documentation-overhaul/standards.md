# Documentation Overhaul - Applicable Standards

## Conventions
- Naming: Deutsche Dateinamen fuer DE Docs, englische fuer EN Docs
- Technische Begriffe bleiben Englisch (CLI, Agent, Workflow, Standards)
- Du-Form in deutschen Docs
- Code-Beispiele immer mit erwarteter Ausgabe
- Markdown: GitHub-Flavored Markdown
- Links: Relative Pfade, keine absoluten URLs fuer interne Referenzen

## Stil
- Professionell-technisch + Tutorial-orientiert
- Jede Seite: Klare H1, Intro-Absatz, dann Inhalt
- Code-Bloecke: Sprache angeben (bash, typescript, yaml, markdown)
- Hinweise: Blockquotes mit Praefix (> **Tipp:** / > **Hinweis:** / > **Warnung:**)
- Keine Emojis ausser in Feature-Tabellen

## Struktur pro Dokument
- H1: Titel
- Intro: 1-2 Saetze was die Seite abdeckt
- Inhaltsverzeichnis bei langen Seiten (optional)
- Abschnitte mit klaren H2/H3
- "Siehe auch" Abschnitt am Ende mit Links zu verwandten Docs
